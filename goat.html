Importantly, your input gene list / gene table must be prepared in a format that is compatible with this tool. In brief,
  • File format: either CSV, TSV, or Excel (.xlsx file, data on the first sheet)
  • Required columns (column names must match exactly)
    1. gene: Human Entrez (NCBI) gene IDs
    2. symbol: gene symbol
    3. effectsize: effect size or log2 foldchange
    4. pvalue: gene p-value (preferably un-adjusted)
    5. signif: was the adjusted p-value significant?
      (boolean values, i.e. 'true' or 'false')
  • No missing/empty values in any of these columns
Further information and examples are available in the documentation.
Instead of preparing and uploading your own gene list, you can immediately try this tool using an example gene list from a proteomics study on Alzheimer's disease by Wingo et al. (download link and reference available through the documentation section)

GOAT analysis

Settings & apply algorithm.

gene score type: highly recommended to set to "effectsize", if available in your gene list
(refer to the Documentation for preparing your input data accordingly)
gene set size, minimum: must be at least 10, recommended to leave at default (10)
gene set size, maximum: threshold on gene set size, i.e. intersection of the input gene set from the selected database and your gene list. Typically set to 1500, recommended to leave at default. The largest allowed gene set size is [50% of the gene list], GOAT will automatically discard larger gene sets
p-value adjustment:
adj. p-value threshold: gene sets with adjusted p-value ≤ this threshold are considered significant. Must be a numeric value between 0 and 1
